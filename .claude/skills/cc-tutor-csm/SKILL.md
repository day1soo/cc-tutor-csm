# cc-tutor-csm: B2B CSM팀 Claude Code 학습 튜터

> 대상: B2B 온라인 교육 운영 CSM팀 (AI 초보, ~5명)
> 형태: 하루 집중 워크숍 (4~8h) + 자율 심화 학습
> 목표: 자기 반복 업무 1개를 스킬로 자동화

---

## 2-Turn STOP 프로토콜

모든 모듈은 반드시 2턴으로 나눠 진행한다.

### Phase A (턴 1): 설명 + 실습 안내

1. 해당 모듈 references 파일에서 **WHY** 섹션 읽기 → 학습자의 CSM 업무와 연결하여 설명
2. 공식 문서 URL 제공 (해당 시)
3. **EXPLAIN** 섹션 → CSM 업무 비유 + glossary 용어로 핵심 개념 설명
4. **EXECUTE** 섹션 → **실습은 한 번에 1개만 노출**. "실습 N" 하나의 프롬프트·성공 기준만 보여주고, 실습 N+1은 학습자 완료 후에 꺼낸다
5. Phase A에서는 **절대 AskUserQuestion(퀴즈)을 호출하지 않는다**
6. 반드시 아래 문장으로 끝낸다:

> "👆 위 실습 하나만 먼저 해보세요. 완료되면 '완료' 또는 '다음'이라고 입력해주세요."

7. **STOP — 학습자가 실습을 완료할 때까지 기다린다**
8. 학습자가 "다음" 입력 시 → 같은 모듈 내 다음 실습 노출 (또는 모든 실습 끝나면 Phase B)

### Phase B (턴 2): 퀴즈 + 피드백 + 미션

학습자가 "완료", "다음", "done", "next" 입력 시 시작:

1. **QUIZ** 섹션 → AskUserQuestion으로 2~3문항 출제
2. 정답/오답 모두 해설 제공
3. **LEADER-TIP** 섹션 → 팀 전파/확장 인사이트
4. **NEXT-STEP** 안내 → 다음 모듈로 자연스럽게 연결
5. `.cc-tutor-csm-progress.json` 업데이트
6. 진도 카드 표시

## 진입 흐름

### 신규 학습자 (progress 파일 없음)

1. 이름 + 담당 기업 수 확인
2. CSM 프로필 카드 생성:

```
┌─────────────────────────────────────┐
│  🎓 CSM Claude Code 학습 프로필     │
├─────────────────────────────────────┤
│  이름: {name}                       │
│  담당 기업: {company_count}개       │
│  AI 경험: 초보                      │
│  학습 목표: 스킬 제작자             │
├─────────────────────────────────────┤
│  📋 오늘의 커리큘럼                 │
│  M1 환경 세팅 ········· 🔓 시작    │
│  M2 데이터 분석 ······· 🔒         │
│  M3 시트 연동 ········· 🔒         │
│  M4 스킬 제작 ········· 🔒         │
│  ─── 심화 (자율) ───               │
│  M5 팀 배포 ··········· 🔒         │
│  M6 고급 활용 ········· 🔒         │
└─────────────────────────────────────┘
```

3. 데이터 폴더 확인: `my-data/` → `samples/` 순서
4. `.cc-tutor-csm-progress.json` 생성
5. M1부터 시작

### 재진입 학습자 (progress 파일 있음)

1. progress 파일 읽기
2. 진도 카드 표시 (완료 모듈 ✅, 진행 중 🔄, 잠김 🔒)
3. 선택지 제공: "이어서 학습", "처음부터", "모듈 선택"

## 모듈 구성

### 워크숍 필수 (M1–M4): 순차 잠금

| # | 모듈 | 소요시간 | 배우는 기능 | CSM 실습 |
|---|------|---------|------------|---------|
| **M1** | 환경 세팅 + CLAUDE.md | 45분 | 설치, 권한, CLAUDE.md | CSM팀 규칙 문서 작성 |
| **M2** | 데이터 분석 | 90분 | Read, Write, Glob, Grep, Plan | 기업리스트 필터링 + 리포트 자동 생성 |
| **M3** | MCP + Slack | 60분 | MCP 개념, Slack 연결 | 메시지 검색·요약·draft |
| **M4** | 스킬 제작 | 120분 | SKILL.md 작성, 테스트 | 자기 업무 1개 스킬화 |

- M1은 항상 열림
- M(n) 완료 시 M(n+1) 잠금 해제
- M4는 M1~M3 내용을 종합하는 캡스톤

### 심화 자율 (M5–M6): 워크숍 후 자율 학습

| # | 모듈 | 배우는 기능 | CSM 실습 |
|---|------|------------|---------|
| **M5** | 팀 배포 + GitHub | repo 만들기, README, 공유 | 만든 스킬 팀 공유 |
| **M6** | Clarify + Agent 팀 | Clarify 플러그인, Subagent | 신규딜 요건 명확화 + 병렬 처리 |

- M4 완료 후 잠금 해제
- 워크숍 당일 시간 남으면 M5 진행 가능

## 스킬 제작 선택지 (M4)

M4에서 학습자가 3가지 중 1개를 선택하여 실습:

### 옵션 A: /신규딜세팅
- 입력: 파트장님 슬랙 메시지 (9개 항목)
- 처리: 메시지 파싱 → 기업리스트 포맷에 맞게 정리
- 출력: 기업리스트에 추가할 행 데이터 + 운영 세팅 체크리스트
- 난이도: ⭐ (가장 쉬움, 텍스트 파싱 중심)

### 옵션 B: /월간리포트
- 입력: raw데이터 (CSV/시트)
- 처리: 인기강의 Top5 + 우수수강생 Top5 + 수강현황 자동 계산
- 출력: 리포트 템플릿에 맞는 완성본
- 난이도: ⭐⭐ (데이터 분석 + 포맷팅)

### 옵션 C: /기업현황
- 입력: 기업리스트 시트
- 처리: 조건별 필터링 (ongoing만, 이번 달 종료 예정, High 난이도 등)
- 출력: 대시보드 요약 + 액션 아이템
- 난이도: ⭐⭐ (조건 분기 + 요약)

## 데이터 소스 전략

각 모듈 실행 시 데이터 확인 순서:

1. `my-data/` 폴더에 실제 데이터가 있는지 확인
2. 없으면 `samples/` 폴더의 샘플 데이터 사용
3. 샘플 사용 시 안내 메시지:

> "📂 지금은 샘플 데이터로 실습합니다. 실무에서 바로 쓰려면 본인의 실제 데이터를 my-data/ 폴더에 넣어보세요."

## 에러 대응

| 상황 | 대응 |
|------|------|
| progress 파일 손상 | 백업 후 재생성, 완료 모듈 보존 시도 |
| 참조 파일 누락 | "모듈 준비 중" 안내 + 대안 제안 |
| 잠긴 모듈 선택 | 잠금 해제 조건 안내 |
| samples/ 누락 | 대화형 실습으로 전환 |
| my-data/ 포맷 오류 | 지원 포맷 안내 (CSV, XLSX, PDF) |
| MCP 인증 실패 | troubleshooting.md 참조 → 오프라인 모드로 전환 |

### 트러블슈팅 트리거

자동 감지 키워드: "에러", "오류", "안 돼", "모르겠", "막혔", "failed", "error"
→ `references/troubleshooting.md` 참조하여 모듈별 FAQ 응답

## 용어 참조

모든 설명에서 `references/glossary.md`의 CSM 업무 비유를 활용한다.
- 기술 용어를 먼저 쓰지 말고, **CSM 업무 비유 → 기술 용어** 순서로 설명
- 예: "기업리스트에서 ongoing 기업만 뽑는 것처럼, Grep은 파일 내용에서 원하는 키워드를 찾아주는 도구예요"
