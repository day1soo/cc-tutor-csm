# CSM 업무 비유 용어집

> 모든 설명에서 CSM 업무 비유를 먼저 쓰고, 기술 용어를 뒤에 붙인다.
> 예: "기업리스트에서 ongoing 기업만 뽑는 것처럼" → Grep 설명

---

## Claude Code 기본

| 기술 용어 | CSM 비유 | 한 줄 설명 |
|-----------|---------|-----------|
| **Claude Code** | 업무 보조 비서가 내 터미널에 상주 | 웹 챗봇(전화 상담)과 달리 내 파일·시트에 직접 접근하는 현장 보조원 |
| **CLAUDE.md** | CSM팀 운영 매뉴얼 | AI가 매번 읽는 규칙서. "수료기준은 이렇게 확인해", "리포트는 이 포맷으로" |
| **권한 모드** | 결재 수준 설정 | "파일 읽기만 OK" ~ "시트 수정까지 OK" 단계별 허용 |
| **터미널** | 명령어로 일하는 작업 공간 | 마우스 클릭 대신 텍스트로 지시하는 환경 |

## 파일 도구 (M2)

| 기술 용어 | CSM 비유 | 한 줄 설명 |
|-----------|---------|-----------|
| **Read** | 기업 시트 열어보기 | 파일을 읽어서 내용 확인 |
| **Write** | 새 리포트 작성 | 새 파일을 처음부터 만들기 |
| **Edit** | 기존 리포트 수정 | 이미 있는 파일의 특정 부분만 바꾸기 |
| **Glob** | 폴더에서 파일 찾기 | "이번 달 리포트 파일 어딨지?" — 이름 패턴으로 검색 |
| **Grep** | 시트에서 특정 기업 검색 | "삼성전자가 들어간 행만 보여줘" — 내용으로 검색 |
| **Plan 모드** | 리포트 작성 전 목차 먼저 잡기 | 실행 전에 계획서를 먼저 보여주고 확인받는 모드 |

## MCP (M3)

| 기술 용어 | CSM 비유 | 한 줄 설명 |
|-----------|---------|-----------|
| **MCP** | 시스템 간 연결 케이블 | Slack, 시트 등 외부 서비스에 Claude Code가 직접 접속하는 통로 |
| **Slack MCP** | 슬랙 열람·검색 권한 | 파트장 메시지·기업 담당자 DM을 채팅 스크롤 없이 바로 검색·요약 |
| **인증(Auth)** | 로그인 | "이 사람이 이 슬랙 워크스페이스에 접근해도 되는지" 확인 과정 |

## 스킬 (M4)

| 기술 용어 | CSM 비유 | 한 줄 설명 |
|-----------|---------|-----------|
| **Skill** | 원클릭 매크로 | "/월간리포트" 한 마디면 리포트가 나오는 자동화 버튼 |
| **SKILL.md** | 매크로 레시피 | 스킬이 어떻게 동작할지 적어놓은 설명서 |
| **트리거** | 실행 키워드 | 스킬을 부르는 명령어 (예: /신규딜세팅) |

## CSM 업무 용어

| 용어 | 설명 |
|------|------|
| **신규딜 세팅** | 새 기업 계약 시 운영 환경 초기 설정 (기업리스트 추가, 시트 생성) |
| **월간 리포트** | 기업 담당자에게 매월 보내는 수강 현황 보고서 |
| **기업리스트** | CSM이 관리하는 전체 기업 운영 현황 마스터 시트 |
| **raw데이터** | 리포트를 만들기 위한 원본 수강 데이터 |
| **수료기준** | 기업별로 다른 교육 완료 조건 (수강률, 수강시간 등) |
| **운영난이도** | 기업 운영의 복잡도 (High/Medium/Low) |
| **독려 발송** | 수강생에게 학습 참여를 유도하는 안내 메일/메시지 |
| **입과** | 수강생이 교육 과정에 등록되는 것 |
