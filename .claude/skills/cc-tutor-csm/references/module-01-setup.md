# 모듈 1: 환경 세팅 + CLAUDE.md

> 배우는 기능: Claude Code 설치 확인, 기본 명령어, CLAUDE.md 시스템 프롬프트
> CSM 실습: CSM팀 운영 규칙 CLAUDE.md 작성
> 소요 시간: 45분

---

## 📌 들어가기 전에 — Claude Code가 뭐예요?

웹 ChatGPT/Claude는 **전화 상담원**이에요. 말로만 소통하고, 내 컴퓨터 파일은 직접 못 봐요.
Claude Code는 **사무실 옆에 앉은 비서**이에요. 내 컴퓨터의 파일을 직접 열고, 수정하고, 새로 만들 수 있어요.

> 👉 그래서 오늘 배우는 CLAUDE.md는 **이 비서에게 쥐여주는 팀 매뉴얼**이에요.

---

## WHY — 왜 CLAUDE.md가 필요한가요?

CSM으로 일하다 보면 AI에게 매번 같은 맥락을 반복 설명하게 돼요.
- "우리 월간 리포트는 3개 섹션이야"
- "수강시간은 HH:MM:SS로 표기해"
- "삼성전자는 수료기준이 달라"

CLAUDE.md는 **비서이 출근하자마자 자동으로 펼쳐보는 팀 매뉴얼**이에요.
한 번 적어두면, 그 다음부터는 따로 설명 안 해도 Claude가 알아서 규칙을 지킵니다.

| Before | After |
|--------|-------|
| 매번 "삼성전자는 수료기준이..." 반복 설명 | CLAUDE.md에 기업별 규칙 기재 → 자동 참조 |
| 리포트 포맷이 사람마다 다름 | 팀 공통 포맷 정의 → 일관된 산출물 |

📖 공식 문서: https://docs.anthropic.com/en/docs/claude-code/overview

---

## EXPLAIN — 핵심 개념 3가지

### 1. CLAUDE.md 구조 (예시)

```markdown
# CSM팀 운영 규칙

## 리포트 포맷
- 월간 운영 레포트는 3개 섹션
- 수강시간 표기: HH:MM:SS

## 기업 분류
- 상태: ongoing / closed
- 운영난이도: High / Medium / Low
```

### 2. 어디에 만들까? — CLAUDE.md 3단계 스코프

| 위치 | 비유 | 적용 범위 |
|------|------|----------|
| 🌍 `~/.claude/CLAUDE.md` (전역) | 회사 취업규칙 | **모든** 프로젝트에 항상 |
| 📁 `{프로젝트폴더}/CLAUDE.md` (프로젝트) | 팀 SOP | 그 폴더에서 실행할 때만 |
| 📄 `{하위폴더}/CLAUDE.md` (서브) | 특정 건 가이드 | 그 하위 폴더에서만 |

적용 순서: 전역 → 프로젝트 → 하위폴더 (아래로 갈수록 덮어씀).

👉 **처음엔 프로젝트 단위부터 시작하세요.** 전역은 다른 업무까지 영향을 주니, 익숙해진 뒤에 공통 규칙만 승급. 오늘 실습은 `cc-tutor-csm/CLAUDE.md`만 손댑니다.

### 3. 권한 모드 (비서에게 얼마나 권한을 줄지)

| 모드 | 비유 | 설명 |
|------|------|------|
| 읽기 전용 | 열람만 OK | 파일 보기만 |
| 일반 | 수정 전 확인 | "이렇게 할까요?" 묻고 진행 |
| 자동 | 전권 위임 | 묻지 않고 바로 실행 (주의!) |

## EXECUTE — 이 대화에서 바로 실행해보세요

> 💡 **지금 상황:** 여러분은 `cc-tutor-csm` 폴더에서 Claude Code를 실행 중이에요. 이 폴더 안에는 학습용 샘플 데이터(`samples/`)와 본인 데이터 자리(`my-data/`)가 있습니다.

---

### Part A: 기본 확인 (10분)

#### 실습 1. 현재 폴더 구조 확인

📋 아래 프롬프트를 입력하세요:
```
지금 작업 중인 cc-tutor-csm 폴더 안에 어떤 파일과 서브폴더가 있는지 보여줘
```

✅ **성공 기준:** `samples/`, `my-data/`, `CLAUDE.md`, `README.md` 등이 보이면 성공!

---

#### 실습 2. 파일 탐색기 대신 Claude로 문서 찾기

> 💡 **이 실습의 포인트**
> 평소 **파일 탐색기 + Ctrl+F**로 하던 걸, Claude한테 한 줄로 시켜보세요.
> CSM 업무에서 제일 흔한 pain point(= 여러 파일에서 특정 용어 찾기)를 Claude Code로 해결하는 감각을 익힙니다.

**Step 1 — 폴더 안에 뭐가 있는지 감 잡기 (2분)**

📋 아래 프롬프트를 입력하세요:
```
samples/ 폴더에 어떤 파일이 있는지 보여주고 각 파일이 뭘 다루는지 한 줄씩 요약해줘
```

✅ **성공 기준:** 6개 파일이 확인됨 — CSV 3개 (기업리스트·리포트raw·체크리스트) + MD 2개 (데이터 추출 SOP·샘플 안내) + TXT 1개 (신규딜 슬랙 메시지)

---

**Step 2 — CSM 용어로 문서 찾기 (5분)**

평소 본인이 자주 쓰는 용어 하나를 떠올려보세요. 그 키워드가 담긴 파일이 어디 있는지 Claude에게 찾아달라고 합니다.

📋 예시 프롬프트 (그대로 써도 되고, **본인 용어로 바꿔서** 해도 됨):
```
samples/ 폴더 안에서 "수료기준"이 언급된 파일을 전부 찾아줘.
각 파일에서 어떻게 언급되는지도 같이 보여줘.
```
```
samples/ 폴더에서 "신규딜" 관련 내용이 담긴 파일 찾아줘.
```
```
samples/ 폴더에서 "독려"라는 단어가 나오는 문서 정리해줘.
```

✅ **성공 기준:** 한 번 질문으로 여러 파일을 가로질러 키워드 위치가 정리되는 경험. "평소 파일 하나씩 열어서 Ctrl+F 하던 것보다 확실히 빠르다"가 체감되면 OK.

---

**Step 3 (선택) — 찾은 내용 종합하기 (3분)**

검색만으로 끝내지 말고, 찾은 내용을 Claude에게 한 번에 정리시켜보세요.

📋 심화 프롬프트 예시:
```
samples/ 폴더 전체에서 "수료기준"이 어떻게 정의돼있는지 
기업별/문서별로 정리해서 표로 보여줘.
```

👉 **체감 포인트:** 평소 여러 파일을 일일이 열어서 복사·붙여넣기로 정리하던 작업이 한 번에 끝남. 이게 바로 Claude Code가 "비서"인 이유.

---

### Part B: CSM팀 CLAUDE.md 작성 (25분)

> 💡 **이미 CLAUDE.md가 있다면?** 이 프로젝트 폴더엔 이미 기본 CLAUDE.md가 있어요. 아래 실습에선 **기존 파일 끝에 추가(append)** 하는 방식으로 진행합니다. 덮어쓰지 않아요.

#### 실습 3. 개인 업무 규칙 설정 — "내 CLAUDE.md" 만들기

> 💡 **핵심 포인트**: CSM팀 공통 규칙은 이미 있으니, 이제 **본인만의 개인 업무 스타일**을 CLAUDE.md에 추가해보세요. 

📋 아래 프롬프트를 입력하세요:
```
CLAUDE.md 끝에 "{이름}님 개인 업무 규칙" 섹션을 새로 추가해줘. 
아래 예시 중 본인 상황에 맞는 것들을 골라서 수정해서 넣어줘:

### {이름}님 개인 업무 규칙
- 담당 기업 수: N개
- 주요 담당 업무: 채용 업무 + 조직문화 업무 (또는 본인 담당 업무)
- 선호하는 데이터 형태: CSV > Excel > PDF 순서
- 리포트 작성 주기: 매월 마지막 주 금요일
- 긴급 이슈 처리: Slack DM 우선 응답
- 무조건 한국어로 문서 작성하기
- 폰트 사이즈는 제목은 20, 부제목은 15, 일반 글은 10
```

**또는 본인만의 실제 업무 규칙이 있으면 그걸 써도 좋아요!**

💡 **실습 예시** (연수님 버전):
```
### 연수님 개인 업무 규칙
- 담당 기업 수: 10개
- 주요 담당 업무: 채용 업무 + 조직문화 업무
- 선호하는 데이터 형태: CSV > Excel > PDF 순서
- 리포트 작성 주기: 매월 마지막 주 금요일
- 긴급 이슈 처리: Slack DM 우선 응답
- 무조건 한국어로 문서 작성하기 
- 폰트 사이즈는 제목은 20, 부제목은 15, 일반 글은 10
```

✅ **성공 기준:** 
- [ ] 기존 CSM팀 공통 규칙은 그대로 유지
- [ ] 본인 이름으로 된 새 섹션이 끝에 추가됨
- [ ] 최소 2줄 이상은 본인 실제 상황에 맞게 수정함
- [ ] 실무에서 자주 겪는 pain point가 1개 이상 반영됨 (예: 한국어 문서, 폰트 규칙, 데이터 선호도 등)

---

#### 실습 3b. Before/After 대비 — CLAUDE.md의 힘 체감하기

CLAUDE.md를 쓰기 **전**에 같은 질문을 해봤으면 비교가 될 텐데 지금은 이미 써둔 상태죠. 대신 **"규칙이 없는 주제"로 질문해서 일반론이 나오는 것**과 **"규칙이 있는 주제"로 질문해서 우리 포맷이 나오는 것**을 비교해보세요.

📋 질문 1 (CLAUDE.md에 규칙 없는 주제):
```
고객사에 보낼 분기별 만족도 조사 설문 문항을 만들어줘
```
→ 일반론적인 설문 문항이 나옴 (Claude가 추측으로 답).

📋 질문 2 (CLAUDE.md에 규칙 있는 주제):
```
월간 운영 리포트 템플릿 하나 만들어줘
```
→ "3개 섹션 / HH:MM:SS / Top 5" 같은 **방금 내가 적은 규칙이 자동 반영**된 템플릿이 나옴.

✅ **성공 기준:** "한 줄 적어둔 게 여기서 작동하는구나"가 체감되면 OK.

---

#### 실습 4. 실전 산출물 생성 + 내 눈으로 검증

이제 CLAUDE.md가 **실제 산출물에 어떻게 반영되는지** 본인이 직접 확인.

📋 아래 프롬프트를 입력하세요:
```
samples/enterprise-list-sample.csv에서 A전자 정보를 찾아서 3월 월간 운영 리포트 초안을 만들어줘. 없는 데이터는 예시 값으로 채우고.
```

✅ **성공 기준 — 본인이 직접 체크 (체크 전 Claude에게 묻지 말기!)**

- [ ] 3개 섹션 구조 (수강 강의 현황 / 수강생 현황 / 운영 사항)
- [ ] 수강시간이 HH:MM:SS
- [ ] Top 5 표
- [ ] 수강 비중(%) 표현

4칸 중 몇 칸 체크됐나요? 3개 이상이면 **"내가 CLAUDE.md에 적은 한 줄이 여기 박혔네"** 감각이 박힌 거예요.

## QUIZ

1. CLAUDE.md의 역할은 무엇인가요?
   - A) Claude Code를 설치하는 설정 파일
   - B) AI가 매번 자동으로 읽는 팀 규칙서 ✅
   - C) 기업 리스트를 저장하는 데이터베이스
   - D) 슬랙 메시지를 보내는 자동화 스크립트
   > 해설: CLAUDE.md는 Claude Code가 대화 시작 시 자동으로 읽어서, 팀 규칙/포맷/용어를 매번 설명하지 않아도 되게 해주는 파일이에요.

2. CLAUDE.md에 넣으면 가장 유용한 내용은?
   - A) 개인 비밀번호
   - B) 오늘 점심 메뉴
   - C) 팀 공통 리포트 포맷과 용어 정의 ✅
   - D) Claude Code 설치 방법
   > 해설: 반복적으로 설명해야 하는 팀 규칙, 포맷, 분류 체계 등을 넣으면 가장 효과적이에요.

3. 이미 CLAUDE.md가 있는 프로젝트에서 새 규칙을 추가할 때 올바른 방식은?
   - A) 기존 파일을 지우고 새로 쓰기
   - B) 기존 내용은 그대로 두고 끝에 추가(append)하기 ✅
   - C) 이름을 CLAUDE2.md로 바꿔서 저장
   - D) 추가 안 해도 Claude가 알아서 기억함
   > 해설: 기존 내용(학습 모드 규칙, 다른 팀 규칙 등)이 날아가지 않도록 append가 원칙이에요.
