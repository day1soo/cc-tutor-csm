# 모듈 2: 데이터 분석 — 기업리스트 + 리포트

> 배우는 기능: Read, Write, Edit, Glob, Grep, Plan 모드
> CSM 실습: 기업리스트 필터링 + 월간 리포트 자동 생성
> 소요 시간: 90분

---

## WHY

지금 월간 리포트를 만드는 과정을 떠올려보세요:

1. 데이터 스튜디오에서 rawdata 다운로드
2. 기간 설정하고 수강시간 0인 행 삭제
3. 피봇 테이블 만들어서 인기 강의 Top 5 뽑기
4. 또 피봇 테이블로 우수 수강생 Top 5 뽑기
5. 초 단위를 HH:MM:SS로 변환
6. 리포트 템플릿에 복사-붙여넣기

**이 6단계를 Claude Code 한 마디로 줄일 수 있어요.**

| Before | After |
|--------|-------|
| 피봇 테이블 수작업 (30분~1시간) | "raw데이터로 3월 리포트 만들어줘" (2분) |
| 기업리스트 눈으로 확인 | "이번 달 종료 예정 기업 보여줘" (10초) |
| 초→시간 변환 수작업 | 자동 변환 |

---

## EXPLAIN

### 1. 파일 도구 6종

| 도구 | CSM 비유 | 언제 쓰나 |
|------|---------|----------|
| **Read** | 시트 열어보기 | 기업리스트, raw데이터 내용 확인 |
| **Write** | 새 리포트 작성 | 분석 결과를 새 파일로 저장 |
| **Edit** | 기존 파일 수정 | 리포트 일부 내용만 수정 |
| **Glob** | 폴더에서 찾기 | "3월 리포트 파일 어딨지?" |
| **Grep** | 내용으로 검색 | "삼성전자가 들어간 행만 찾아줘" |
| **Plan** | 작업 전 계획 확인 | "이렇게 할 건데 맞아?" 확인 후 실행 |

### 2. Plan 모드

복잡한 작업(리포트 생성 등)은 **Plan 모드**로 시작하는 게 안전해요.
마치 리포트 작성 전에 목차를 먼저 잡는 것과 같아요.

```
Plan 모드 ON → Claude가 계획서 보여줌 → 확인 후 실행
```

## EXECUTE

> 💡 **모든 실습 공통 팁**
> 아래 실습은 `samples/` 폴더의 예시 파일로 진행하지만, **본인이 실제로 다루는 CSV/엑셀/텍스트 파일이 있으면 언제든 그걸로 해봐도 됩니다.** 
> - 방법 1: 본인 파일을 `my-data/` 폴더에 복사해서 넣기 → 프롬프트에서 `samples/...` 대신 `my-data/파일명` 쓰기
> - 방법 2: 프롬프트에 본인 파일의 **전체 경로를 그대로 붙여넣기** (예: `C:\Users\...\내자료.csv`)
> 
> 본인 데이터로 실습하면 **워크숍 끝나자마자 실무에 바로 적용**할 수 있어요.

### Part A: 기업리스트 분석 (30분)

**실습 1.** 기업리스트를 읽고 현황을 파악하세요.

📋 아래를 입력하세요:
```
samples/enterprise-list-sample.csv 읽고 총 기업 수 / ongoing:closed 비율 / 운영난이도 분포 요약해줘
```

**실습 2.** 특정 조건으로 필터링해보세요.

> 💡 **예시는 ongoing + High이지만 본인이 평소 자주 보는 조건으로 바꿔도 OK.**
> (예: 내가 담당하는 CSM만 / 이번 주 입과 기업만 / 수료기준이 복합조건인 기업만)

📋 아래를 입력하세요:
```
기업리스트에서 [본인 조건] 기업만 뽑아서 표로 보여주고 몇 개인지 알려줘
```

**실습 3.** 이번 달 주의가 필요한 기업을 찾아보세요.

📋 아래를 입력하세요:
```
기업리스트에서 앞으로 30일 안에 강의종료일자가 지나는 ongoing 기업 찾아줘
```

### Part B: 월간 리포트 자동 생성 (45분)

**실습 4.** raw데이터를 읽어보세요.

📋 아래를 입력하세요:
```
samples/report-rawdata-sample.csv 읽고 데이터 기간 / 총 수강 건수 / 컬럼 구성 요약해줘
```

**실습 5.** Plan 모드로 리포트 생성 계획을 세우세요.

> 💡 **Plan 모드가 뭐예요?**
> Claude가 바로 실행하지 않고, **"이렇게 할 건데 맞아요?"** 계획서를 먼저 보여주는 안전장치예요. 리포트처럼 단계가 많은 작업은 계획 검토 후 실행이 안전합니다.

> 🎯 **이 실습의 숨은 메시지**
> 프롬프트가 한 줄인데도 CLAUDE.md에 적어둔 3섹션·Top 5·HH:MM:SS가 **자동 반영된 계획서**가 나오면 성공. 이게 M1에서 CLAUDE.md를 쓴 이유.

📋 아래를 입력하세요:
```
Plan 모드로, CLAUDE.md 참고해서 samples/report-rawdata-sample.csv로 
월간 운영 리포트 만들어줘.
```

→ 계획 확인 후 "실행해줘"로 진행

✅ **성공 기준:** 계획서에 CLAUDE.md의 리포트 작성 규칙(3섹션/Top 5/HH:MM:SS/수강비중%)이 **프롬프트에 명시하지 않았는데도** 반영됨

**실습 6.** 완성된 리포트를 파일로 저장하세요.

📋 아래를 입력하세요:
```
방금 만든 리포트를 report-output.md 파일로 저장해줘
```

> 💡 **다음 M3 맛보기**
> - 지금은 `.md` 파일로 저장했지만 → M3에선 슬랙 메시지도 직접 검색·요약 가능
> - CSM의 일상(파트장 신규딜 DM, 기업 담당자 메시지)이 Claude 한 마디로 정리됨
> - 이번 실습 6 결과에 "근데 내 실무는 시트/슬랙에 있는데" 느꼈다면 M3가 답

### Part C: Before vs After 체감 (15분)

**실습 7.** 수작업 SOP와 직접 비교해보세요.

📋 아래를 입력하세요:
```
samples/data-extraction-sop.md 읽어줘
```

✍️ Claude 답을 보고 **본인이 직접 답해보세요** (Claude에게 물어보지 말고):

- 이 SOP의 몇 단계가 실습 5의 한 줄 프롬프트로 사라졌나요?
- 숙련자 기준 30분 → 지금 몇 초? 본인 체감으로 적어보세요.
- 담당 기업 20개 기준 월 몇 시간이 확보되나요?

💡 답을 적은 뒤에 Claude에게 "내 계산이 맞는지 확인해줘" 한 번 더 던져도 OK.

## QUIZ

1. 기업리스트에서 "ongoing 상태의 High 난이도 기업"만 찾으려면 어떤 도구 조합이 적절한가요?
   - A) Read만 쓰면 됨
   - B) Read로 파일 읽기 + 조건 필터링 요청 ✅
   - C) Glob으로 검색
   - D) Write로 새 파일 생성
   > 해설: Read로 파일을 열고, Claude에게 조건을 말하면 필터링해줘요. Grep은 텍스트 검색, Glob은 파일명 검색이라 이 경우엔 Read + 조건 지시가 맞아요.

2. 리포트 만들기처럼 복잡한 작업 전에 Plan 모드를 쓰는 이유는?
   - A) 속도가 빨라져서
   - B) 실행 전에 계획을 확인하고 수정할 수 있어서 ✅
   - C) 파일을 자동으로 백업해줘서
   - D) MCP가 필요 없어져서
   > 해설: Plan 모드는 "이렇게 할 건데 맞아?" 확인 후 실행하는 안전장치예요. 리포트 포맷이 틀리면 다시 하기 번거로우니까, 미리 계획을 검토하는 거예요.

3. raw데이터의 수강시간이 초(seconds) 단위일 때, HH:MM:SS로 변환하려면?
   - A) 엑셀에서 직접 수식을 만들어야 함
   - B) Claude에게 "수강시간을 HH:MM:SS 형식으로 변환해줘"라고 말하면 됨 ✅
   - C) 별도 프로그램 설치가 필요함
   > 해설: Claude Code는 데이터 변환을 자연어로 지시하면 처리해요. 피봇 테이블 수작업도, 시간 변환도 전부 말로 요청하면 돼요.
