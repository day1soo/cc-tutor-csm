# 모듈 2: 데이터 분석 — 핵심 2가지만 체험

> 배우는 기능: Read, Grep, Plan 모드 (핵심만)
> CSM 실습: 기업리스트 필터링 + 월간 리포트 자동 생성
> 소요 시간: 45분 (실습 2개로 압축)

---

## WHY

지금 월간 리포트를 만드는 과정을 떠올려보세요:

1. 데이터 스튜디오에서 rawdata 다운로드
2. 기간 설정하고 수강시간 0인 행 삭제
3. 피봇 테이블 만들어서 인기 강의 Top 5 뽑기
4. 또 피봇 테이블로 우수 수강생 Top 5 뽑기
5. 초 단위를 HH:MM:SS로 변환
6. 리포트 템플릿에 복사-붙여넣기

**이 6단계를 Claude Code 한 마디로 줄일 수 있어요.**

| Before | After |
|--------|-------|
| 피봇 테이블 수작업 (30분~1시간) | "raw데이터로 3월 리포트 만들어줘" (2분) |
| 기업리스트 눈으로 확인 | "이번 달 종료 예정 기업 보여줘" (10초) |
| 초→시간 변환 수작업 | 자동 변환 |

## EXPLAIN

### 1. 파일 도구 6종

| 도구 | CSM 비유 | 언제 쓰나 |
|------|---------|----------|
| **Read** | 시트 열어보기 | 기업리스트, raw데이터 내용 확인 |
| **Write** | 새 리포트 작성 | 분석 결과를 새 파일로 저장 |
| **Edit** | 기존 파일 수정 | 리포트 일부 내용만 수정 |
| **Glob** | 폴더에서 찾기 | "3월 리포트 파일 어딨지?" |
| **Grep** | 내용으로 검색 | "삼성전자가 들어간 행만 찾아줘" |
| **Plan** | 작업 전 계획 확인 | "이렇게 할 건데 맞아?" 확인 후 실행 |

### 2. Plan 모드

복잡한 작업(리포트 생성 등)은 **Plan 모드**로 시작하는 게 안전해요.
마치 리포트 작성 전에 목차를 먼저 잡는 것과 같아요.

```
Plan 모드 ON → Claude가 계획서 보여줌 → 확인 후 실행
```

## EXECUTE

> 💡 **실습 2개로 압축한 이유**
> 워크숍에서 **핵심만 체험**해서 CSM들이 어떤 걸 할 수 있는지 파악하는 게 목표. 너무 많은 실습보다는 **임팩트 있는 2가지**로 승부!

> 💡 **본인 데이터로도 가능**
> `samples/` 파일 대신 **본인 실제 CSV/엑셀 파일**을 `my-data/` 폴더에 넣거나 전체 경로로 지정해서 실습 가능. 워크숍 끝나자마자 실무 적용!

### 실습 1: 조건부 기업 필터링 — "이번 달 위험 기업 찾기" (15분)

> 🎯 **체험 포인트**: Read + Grep 조합으로 복잡한 조건 검색이 한 방에 끝남

📋 **아래 프롬프트를 입력하세요**:
```
samples/enterprise-list-sample.csv에서 ongoing이면서 앞으로 30일 안에 강의종료일자가 지나는 기업을 찾아서 표로 정리해줘. 남은 일수도 계산해서 포함해줘.
```

✅ **성공 기준**: 
- ongoing 상태 기업만 필터링
- 30일 이내 종료 예정 기업만 추출
- 남은 일수 자동 계산
- 표 형태로 깔끔하게 정리

💡 **실무 활용**: 매주 월요일 위험 기업 체크가 10초로 단축!

---

### 실습 2: Plan 모드로 월간 리포트 자동 생성 — "CLAUDE.md 마법 확인" (30분)

> 🎯 **체험 포인트**: 한 줄 프롬프트로 M1에서 설정한 규칙들이 자동 반영되는 마법 같은 경험

**Step 1 — Plan 모드로 계획 세우기**

📋 **아래 프롬프트를 입력하세요**:
```
Plan 모드로, samples/report-rawdata-sample.csv를 사용해서 CLAUDE.md에 정의된 형식에 맞는 월간 운영 리포트를 만들어줘
```

→ Claude가 계획서를 보여줄 거예요. **프롬프트에 명시하지 않았는데도** 3섹션/Top 5/HH:MM:SS/수강비중% 규칙이 들어있는지 확인!

**Step 2 — 계획 확인 후 실행**

📋 **계획이 마음에 들면 입력하세요**:
```
실행해줘
```

✅ **성공 기준**: 
- [ ] 계획서에 CLAUDE.md 규칙들이 자동 반영됨
- [ ] 3개 섹션 구조 (수강 강의 현황/수강생 현황/운영 사항)
- [ ] Top 5 형식
- [ ] 수강시간 HH:MM:SS 표기
- [ ] 수강 비중(%) 자동 계산
- [ ] 연수님 개인 규칙 반영 (한국어 문서, 폰트 설정 등)

💡 **핵심 메시지**: 이게 바로 **"M1에서 CLAUDE.md를 설정한 이유"**예요! 한 번 설정으로 모든 후속 작업이 자동화됩니다.

   - B) 실행 전에 계획을 확인하고 수정할 수 있어서 ✅

   - A) Claude가 기본으로 알고 있는 포맷이어서
   - B) CLAUDE.md를 Claude가 대화 시작 시 자동으로 읽기 때문에 ✅
