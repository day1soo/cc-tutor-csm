# 모듈 3: MCP + Slack 연동

> 배우는 기능: MCP 개념, Slack MCP 연결, 채널·DM 검색·요약
> CSM 실습: 파트장 신규딜 메시지 / 기업 담당자 DM 바로 검색
> 소요 시간: 60분

---

## WHY

지금 슬랙에서 뭔가 찾을 때:

- 채널을 위아래로 스크롤하며 눈으로 훑기
- 여러 채널·DM을 오가며 맥락 이어 붙이기
- "파트장님이 어제 뭐라고 하셨더라..." 다시 검색

**MCP(연결 케이블)를 꽂으면 Claude가 슬랙을 대신 훑어줍니다.**

| Before | After |
|--------|-------|
| 채널 스크롤 + Ctrl+F 반복 | "최근 7일 '신규딜' 메시지 찾아줘" 한 줄 |
| 여러 메시지 맥락을 머리로 조립 | "이 스레드 3줄 요약해줘" 바로 끝 |

---

## EXPLAIN

### 1. MCP란?

> Claude Code = 사무실의 비서
> MCP = 비서에게 준 시스템 접근 권한
> Slack MCP = "슬랙 열람·검색 가능" 권한

- MCP 없이: Claude는 내 컴퓨터 파일만 본다
- MCP 연결: 슬랙·Notion 등 외부 서비스도 직접 접근

### 2. Slack MCP로 할 수 있는 것

- **검색**: 채널·DM에서 키워드로 메시지 찾기
- **읽기**: 특정 스레드 / 특정 기간 메시지 불러오기
- **Draft**: 답장 초안 작성 (⚠️ 학습 모드에선 **실제 발송 금지**)

---

## EXECUTE

> 💡 본인 실제 슬랙 워크스페이스로 실습합니다. 퍼실리테이터가 준비한 공용 시트·ID 같은 건 없어요.

---

### 실습 1. Slack MCP 연결 (15분)

**Step 1 — Claude에게 연결 요청**

📋 아래를 입력하세요:
```
Slack MCP 연결하고 싶어. 인증 시작해줘
```

→ Claude가 authorization URL을 알려줍니다.

**Step 2 — 브라우저에서 승인**

- 알려준 URL을 브라우저에 붙여넣기
- 본인 Slack 워크스페이스 선택 → 권한 허용
- 브라우저가 `http://localhost:.../callback?code=...` 로 리디렉트 (페이지 안 열려도 OK)

**Step 3 — 콜백 URL을 Claude에게 전달**

- 브라우저 **주소창 URL 전체를 복사**
- 아래처럼 입력:

```
콜백 URL 여기 있어: [주소창에서 복사한 URL]
```

✅ **성공 기준:** "Slack 연결 완료" 메시지 + 이후 실습에서 메시지 검색 가능

❓ **막히면:** troubleshooting.md 참조 또는 퍼실리테이터 호출

---

### 실습 2. 본인 키워드로 메시지 검색 (15분)

평소 슬랙에서 자주 찾는 키워드 하나를 떠올려보세요.

📋 아래를 입력하세요 (키워드는 본인 걸로):
```
Slack에서 최근 14일 안의 "신규딜" 포함 메시지 찾아줘
```

💡 **다른 키워드 예시**: 본인 담당 기업명 / "수료기준" / "독려"

✅ **성공 기준:** 여러 채널의 메시지가 한 번에 리스트업

---

### 실습 3. 검색 결과 요약 (15분)

방금 찾은 메시지를 Claude가 한 줄로 정리해줍니다.

📋 아래를 입력하세요:
```
방금 찾은 메시지들을 채널별로 / 날짜순으로 3줄 요약해줘
```

✅ **성공 기준:** 여러 메시지 맥락이 하나로 정리됨 — "아, 이거 스크롤 안 해도 되네"

---

### 실습 4. (선택) Draft 메시지 작성 (15분)

> ⚠️ **학습 모드 규칙:** Slack 메시지 **실제 발송 금지**. draft(초안)만.

📋 아래를 입력하세요:
```
방금 요약 내용을 기반으로, 파트장님께 답장할 draft 메시지 만들어줘. 보내지는 말고 초안만.
```

✅ **성공 기준:** 복붙해서 바로 쓸 만한 초안이 나옴

## QUIZ

1. MCP의 역할을 가장 잘 설명한 것은?
   - A) Claude Code를 똑똑하게 만드는 업그레이드
   - B) Claude Code가 외부 서비스(Slack 등)에 직접 접근하게 해주는 연결 통로 ✅
   - C) 슬랙을 자동 백업해주는 도구
   > 해설: MCP는 "연결 케이블". 한 번 꽂으면 Claude가 슬랙을 바로 읽고 쓸 수 있어요.

2. Slack MCP 연결은 워크숍 당일 누가 하나요?
   - A) 퍼실리테이터가 미리 세팅
   - B) 각자 본인 워크스페이스 계정으로 현장에서 ✅
   - C) IT팀 신청 필요
   > 해설: 개인 Slack 계정으로 현장에서 바로 연결해요. 사전 준비 불필요.

3. 학습 모드에서 Slack MCP로 할 수 있는 것 / 없는 것은?
   - A) 메시지 발송까지 전부 가능
   - B) 검색 / 읽기 / draft 작성만 OK, 실제 발송은 금지 ✅
   - C) 읽기만 OK, draft도 금지
   > 해설: CLAUDE.md 학습 모드 규칙. 실제 발송은 워크숍 외부에서 본인 판단으로.

## LEADER-TIP

> 💡 **슬랙이 열리는 순간 CSM 업무 30%가 줄어든다**
>
> - 파트장 신규딜 메시지 → M4에서 만들 `/신규딜세팅` 스킬로 바로 투입
> - 기업 담당자 DM 스레드 → "이 고객이 최근 뭘 요청했지?" 한 번에 정리
> - 여러 채널 공지 → 월간 회고에 쓸 재료 자동 수집
>
> 팀원 5명이 각자 Slack MCP를 연결하면, **각자 본인 상황에 맞게** 스킬을 확장할 수 있어요.
