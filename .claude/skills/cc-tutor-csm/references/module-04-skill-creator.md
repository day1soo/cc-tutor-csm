# 모듈 4: 스킬 제작 — 내 업무 자동화

> 배우는 기능: SKILL.md 설계·작성·저장·테스트
> CSM 실습: 본인 반복 업무 1개를 빈칸 템플릿으로 풀어내 스킬로 전환
> 소요 시간: 120분

---

## WHY

M1~M3에서 배운 걸 매번 길게 지시할 건가요?

- 한 줄 명령 (`/기업현황`) → M1+M2+M3 합작 결과가 쏟아짐
- 스킬 = "**매번 반복하는 지시를 레시피로 저장**"
- 요리할 때 레시피 검색 vs 즐겨찾기 차이

---

## EXPLAIN

### 1. 좋은 스킬의 3요소

- **명확한 입력** — 뭘 넣는지 (예: raw데이터 CSV 경로)
- **정해진 처리** — 매번 같은 방식 (예: Top 5 추출 + HH:MM:SS 변환)
- **일관된 출력** — 결과 포맷 고정 (예: CLAUDE.md 리포트 양식)

### 2. SKILL.md 최소 구조

```yaml
---
name: 스킬이름
description: 이 스킬이 뭘 하는지 한 줄
triggers: /명령어, 별칭
---

## 입력 / 처리 단계 / 출력
## 규칙
```

### 3. 📂 스킬 파일은 어디에 저장?

- Claude Code는 `.claude/skills/{스킬이름}/SKILL.md` 를 슬래시 명령어로 인식
- 예: `.claude/skills/신규딜세팅/SKILL.md` 저장 → `/신규딜세팅` 호출 가능
- 💡 팁: "SKILL.md를 **올바른 경로에 저장해줘**" 한 마디면 Claude가 폴더까지 만들어줌

---

## EXECUTE

### Part A: 자동화할 내 업무 고르기 (10분)

**실습 1.** 본인이 **매주/매월 반복하는 업무 1개**를 골라서 아래 빈칸을 채워보세요.

```
자동화할 업무: ________________________________

입력 (뭐가 있어야 시작 가능?):
  ________________________________

처리 단계 (3~5개로 쪼개서):
  1. ________________________________
  2. ________________________________
  3. ________________________________
  4. (필요하면) ________________________________
  5. (필요하면) ________________________________

출력 (결과물 포맷):
  ________________________________
```

> 💡 아이디어가 안 떠오르면 아래 3개 옵션 중 하나를 골라도 OK.

<details>
<summary>📋 막힐 때 참고할 3가지 옵션</summary>

| 옵션 | 스킬명 | 난이도 | 요약 |
|------|--------|--------|------|
| A | `/신규딜세팅` | ⭐ | 파트장 슬랙 메시지 → 기업리스트 행 데이터 |
| B | `/월간리포트` | ⭐⭐ | raw데이터 → 완성 리포트 |
| C | `/기업현황` | ⭐⭐ | 기업리스트 → 조건 필터 + 액션 아이템 |

옵션별 빈칸 예시는 이 문서 맨 아래 **레퍼런스 박스** 참조.
</details>

✅ **성공 기준:** 빈칸 5개(업무/입력/처리/출력)가 본인 언어로 채워졌다 — 꼭 완벽할 필요 없음

---

### Part B: 빈칸 → SKILL.md 초안 (15분)

**실습 2.** 방금 채운 빈칸을 Claude에게 넘기세요.

📋 아래를 입력하세요:
```
내가 적은 내용으로 SKILL.md 초안 만들어줘. 파일은 .claude/skills/{스킬이름}/SKILL.md 경로에 저장해줘
```

Claude가 폴더 생성 + SKILL.md 작성까지 합니다.

✅ **성공 기준:** `.claude/skills/` 아래 내 스킬 폴더가 생성됐고, SKILL.md 파일이 있다

---

### Part C: 테스트 + 개선 (45분)

**실습 3.** 만든 스킬을 테스트하세요.

📋 본인 스킬 이름으로 호출:
```
/{내 스킬 이름}

[필요한 입력 데이터 또는 파일 경로]
```

> 💡 **옵션 A를 골랐다면**: 테스트 데이터로 `samples/new-deal-message-sample.txt` 사용
>    또는 **본인 슬랙에서 실제 파트장 메시지**를 가져와서 넣어보세요 (M3 연계):
>    `Slack에서 최근 '신규딜' 메시지 찾아서 /신규딜세팅에 넣어줘`

**실습 4.** 결과를 **본인이 직접** 검증하세요.

✍️ 아래 3개 자가 체크:

- [ ] 결과가 내 실무 포맷과 일치하나?
- [ ] 빠진 정보가 있나? 어디가 빠졌나?
- [ ] 운영난이도·분류 자동 판단이 있다면, 내 기준과 맞나?

위 중 하나라도 "아니" 라면, SKILL.md를 Claude에게 지목해서 고쳐달라고:
```
SKILL.md의 [문제 부분]을 [내 기준]으로 바꿔줘
```

**실습 5.** 개선된 스킬 재테스트 → 같은 /{스킬} 호출

## 📦 레퍼런스 박스 — 옵션 A/B/C 완성 예시

> Part A에서 막혔을 때 참고. 그대로 써도 되고, 본인 상황에 맞게 수정해도 됨.

<details>
<summary>옵션 A: /신규딜세팅</summary>

- **입력**: 파트장 신규딜 슬랙 메시지 (9개 항목)
- **처리**:
  1. 메시지에서 9개 항목 추출 (기업명·수강기간·과정포맷·운영과정정보·수료기준·대상명단·기업담당자·독려발송여부·기타특이사항)
  2. 빠진 항목은 "⚠️ 누락" 표시 (추측 금지)
  3. 과정포맷 → 플랜 매핑 (구독제→올, 싱글→싱글, 수강접수→싱글-접수)
  4. 운영난이도 자동 판단 (수강률만→Low, 시간+기간→Medium, 복합조건→High)
  5. 기업리스트 행 포맷으로 출력
- **출력**: 기업리스트 추가용 1행 + 누락 확인 메시지 초안
- **규칙**: CLAUDE.md 신규딜 9개 항목 기준 검증 / 추측 금지

</details>

<details>
<summary>옵션 B: /월간리포트</summary>

- **입력**: raw데이터 CSV 경로 / 기업명 / 대상 월 / 총 수강인원
- **처리**:
  1. 대상 월 기간 필터링 + 수강시간 0/공란 제거
  2. 인기 강의 Top 5 (수강시간 SUM)
  3. 인기 강의 Top 5 (수강인원 COUNTA)
  4. 우수 수강생 Top 5 (수강시간 SUM)
  5. 수강시간 → HH:MM:SS 변환
- **출력**: CLAUDE.md 리포트 양식(3섹션) + 운영 사항은 수기 영역으로 비워두기
- **규칙**: 총 수강인원은 사용자 입력 필수 (rawdata로 계산 불가)

</details>

<details>
<summary>옵션 C: /기업현황</summary>

- **입력**: 기업리스트 CSV 경로 / 기준일 (기본 오늘)
- **처리**:
  1. 전체 현황 (ongoing:closed / CSM별 / 난이도 분포)
  2. 주의 필요 리스트 (이번 달 종료 / 다음 달 종료 / High 난이도 ongoing)
  3. 액션 아이템 (종료 임박 → 담당자 연장 확인 / High → 수료율 확인)
- **출력**: 대시보드 요약 + 액션 아이템 리스트 (담당자·이메일 포함)
- **규칙**: closed는 주의 리스트 제외 / 날짜는 기준일 기준

</details>
