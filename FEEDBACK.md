- **문제:** "Claude Code란?" (전화 상담원 vs 인턴 비유)이 EXPLAIN에 있는데, 이게 WHY보다 뒤에 나온다.

- **수정 방향:** "Claude Code = 사무실 옆 인턴" 비유를 **WHY 섹션의 맨 앞**으로 옮긴다. 또는 WHY 위에 "Claude Code란?" 짧은 박스를 별도 배치. 그 다음에 "그래서 이 인턴에게 CLAUDE.md(팀 매뉴얼)를 쥐여주는 거예요"로 자연스럽게 연결.

### [2026-04-21] 실습 전반이 "복붙 드릴"이 돼버림 (Part A·B 공통)
- **문제:** 실습 1~4 모두 "아래 프롬프트를 그대로 입력하세요"로 구성됨. 학습자는 시키는 대로 복사·붙여넣기만 하게 되고, 이 도구로 **무엇을 할 수 있는지** 체감하지 못한다.
- **왜 문제인가:** 예를 들어 실습 2(`enterprise-list-sample.csv` 요약)는 Claude가 대신 답을 주고 끝남. 학습자가 그 파일을 열어보고 "어? 이 데이터로 뭘 물어볼 수 있지?"를 스스로 떠올리는 기회가 없다. 워크숍 끝나고 "결국 프롬프트 복붙만 하다 왔네" 하고 돌아갈 위험.
- **수정 방향:**
  1. 각 실습을 **2단계 구조**로: (1) 먼저 학습자가 파일을 열어서 내용을 훑어보게 한다 (예: "샘플 CSV를 Excel이나 VSCode로 열어보세요. 어떤 컬럼이 있고 몇 행이 있나요?"). (2) 그 다음에 "이 데이터로 궁금한 걸 Claude에게 자유롭게 질문해보세요" — 예시 프롬프트 2~3개 제시하되 필수는 아님.
  2. 실습 2를 구체적으로 고치면: "enterprise-list-sample.csv를 직접 열어보고, 그 안에서 **본인이 궁금한 것 하나를 골라** Claude에게 물어보세요. 예: 'ongoing 상태 기업만 몇 개야?', '운영난이도 High인 기업 이름 알려줘'" 같이 **질문 설계를 학습자에게 맡긴다**.
  3. 이래야 "아, 내가 평소 수작업으로 하던 집계·필터가 한 줄로 해결되네"가 체감됨 → Claude Code의 효용이 와닿는다.

### [2026-04-21] 실습 4 — 메타 질문을 실전 산출물 + 반영 요약으로 교체
- **문제:** 원본 실습 4는 `어떤 포맷으로 만들어야 하는지 CLAUDE.md 기준으로 설명해줘` → Claude가 CLAUDE.md를 요약해서 되뱉는 **메타 확인**에 그침. 규칙이 **실제로 적용되는 장면**을 학습자가 못 본다.
- **왜 문제인가:** "오 Claude가 내 규칙 알고 있네" 정도의 얕은 확인. CSM이 실무에서 느낄 효용(= 반복 설명 없이 규칙 반영된 산출물이 나옴)을 체감 못 함.
- **수정 방향:**
  1. **프롬프트 교체:** `samples/enterprise-list-sample.csv에서 A전자 정보를 찾아서, 3월 월간 운영 리포트 초안을 만들어줘.` → Claude가 규칙을 **적용해서** 실제 리포트 초안을 산출. "3섹션 / HH:MM:SS / Top 5 / 수강 비중(%)"이 결과물에 자연스럽게 반영되는 게 눈에 보임.
  2. **CLAUDE.md 반영 요약 박스 추가:** 산출물 바로 아래에 "✅ CLAUDE.md 어떤 규칙이 반영됐는지" 표를 붙임 — 예: `3개 섹션 구조 ← 리포트 작성 규칙 / HH:MM:SS ← 리포트 작성 규칙 / Top 5 ← 리포트 작성 규칙 / 상태: ongoing ← 기업 운영 분류`. 학습자가 "한 줄 적은 게 여기 반영됐구나"를 직관으로 확인.
  3. (선택) 더 강하게 하려면 Before/After 비교 추가 — CLAUDE.md 없이 같은 질문 먼저 → 일반론 답변 → CLAUDE.md 켜고 재질문 → 규칙 반영 답변. 하지만 시간 부담 있으면 요약 박스만으로도 충분.

### [2026-04-21] 실습 3 — 규칙 내용이 "강요"돼서 학습자 맞춤이 안 됨
- **문제:** 실습 3 프롬프트에 "리포트 3섹션 / HH:MM:SS / 신규딜 9항목"이 **완성된 형태로 박혀있음**. 학습자는 이걸 그대로 복붙하게 되고, 결국 "내 팀 규칙"이 아니라 "튜터가 준 규칙"을 넣게 된다.
- **왜 문제인가:**
  1. CSM팀마다 리포트 포맷·분류 체계가 다를 수 있는데 동일 내용 복붙은 개인화 경험을 망침.
  2. "이 규칙을 넣으면 실제로 **어떤 효과**가 나는지"가 안 보임 → CLAUDE.md의 힘을 체감 못 함. 실습 4가 있긴 하지만 "포맷 설명" 수준이라 약함.
- **수정 방향:**
  1. **예시로 제시**하고 **커스터마이즈 유도**: "아래는 예시예요. 본인 팀 규칙으로 바꿔도 되고, 그대로 써도 돼요. 최소 1줄은 본인 상황에 맞게 수정/추가해보세요." → 학습자가 **자기 것**으로 만든다.
  2. **Before/After 대비 시연**: 실습 3 직후에 "CLAUDE.md 없이 vs 있이" 같은 질문을 두 번 던져서 결과 차이를 직접 보게 한다. 예:
     - (없이) "월간 리포트 템플릿 하나 만들어줘" → Claude가 일반적인 포맷 제시
     - (있이) 같은 질문 → 방금 적은 "3섹션 / HH:MM:SS / Top 5" 규칙이 자동 반영된 결과
     → "아! CLAUDE.md에 한 줄 적었더니 Claude가 그걸 기억하네" 가 체감됨.
  3. 실습 4는 "포맷 설명"에서 "**실제 산출물 생성**"으로 업그레이드 (예: "A전자 3월 리포트 **초안**을 만들어줘" → 규칙이 반영된 실제 리포트가 나오는지 확인).

### [2026-04-21] 실습 3 — 기존 CLAUDE.md 있을 때 분기 안내 누락

  3. 추가로: 실습 4 테스트 시 "기존 규칙 + 새 CSM 규칙이 **둘 다** 참조되는지" 확인 포인트를 넣으면 좋음.

### [2026-04-22] 실습 5 직후 "시트로도 돼?" 질문이 자연스럽게 나옴 — M2/M3 전환 재설계 필요
- **관찰:** 학습자(연수님)가 리포트 초안을 받자마자 **"이걸 구글 시트로도 만들어줄 수 있어?"** 라고 질문. 이게 복수 학습자에게도 반복될 가능성 매우 높음.
- **왜 일어나나:** CSM 실무는 대부분 **구글 시트 기반**. `.md` 파일로 저장하는 건 학습 목적엔 맞지만 실무 연속성 측면에선 어색함 — "이걸 또 시트에 옮겨야 해?" 하는 추가 수작업 느낌.
- **현재 구조의 문제:**
  1. M2 실습 6이 `.md` 파일 저장으로 끝남 → 체감상 "반쪽 결과물"
  2. M3(MCP + 시트)는 M2 완전히 끝난 다음 시작 → 학습자는 M2 동안 "왜 시트로 안 하지?" 의문 품음
  3. M2와 M3 사이 단절감이 큼
- **수정 방향 (선택지):**
  - **옵션 A (권장):** M2 실습 6에 "다음 M3에서 이걸 시트로 바로 쏘는 법 배울 거예요" **티저** 추가 + M2-M3 전환 스토리라인 강화. `.md` → 시트 자동 전송이 M3의 킬러 실습.
  - **옵션 B:** M2 실습 6을 "파일 저장 또는 시트 저장 중 선택"으로 분기. 시트 선택 시 M3 프리뷰 역할. 단, 사전 세팅 복잡도 증가.
  - **옵션 C:** 학습용 시트 ID를 **워크숍 D-1에 퍼실리테이터가 세팅**하는 체크리스트 강화 (SKILL.md 사전 준비에 이미 있지만 더 강조). 없으면 M3 전체 막힘.
- **반영 위치:** `module-02-file-tools.md` 실습 6, `module-03-mcp-sheets.md` 서두, `SKILL.md` 사전 준비 체크리스트.

### [2026-04-22] 실습 5 프롬프트가 지나치게 장황 — CLAUDE.md 취지에 역행
- **문제:** 현재 실습 5 프롬프트는 "CLAUDE.md 포맷대로 + 1.인기강의 Top 5 + 2.수강 현황 + 3.우수수강생 Top 5..." 식으로 **CLAUDE.md에 이미 적힌 내용을 또 프롬프트에 나열함**. 
- **왜 문제인가:** M1에서 CLAUDE.md를 배운 핵심 가치 = "반복 설명 안 해도 됨". 그런데 실습 5가 **프롬프트에서 그 규칙을 다시 풀어씀** → 학습자 입장에선 "CLAUDE.md 있는 게 무슨 의미야?" 라는 의문 생김.
- **수정 방향:** 프롬프트를 **극단적으로 간결**하게: 
  ```
  CLAUDE.md 참고해서 samples/report-rawdata-sample.csv로 월간 운영 리포트 만들어줘.
  Plan 모드로 먼저 계획 보여주고.
  ```
- **교육적 효과:** 한 줄로 시켰는데 CLAUDE.md에 적힌 3섹션·Top 5·HH:MM:SS가 자동 반영된 계획서가 나오면 → **"아, 이래서 CLAUDE.md를 쓰는구나"** 가 체감됨. M1의 가치가 M2에서 증명됨.
- **반영 위치:** `module-02-file-tools.md` 실습 5 프롬프트 교체. M3/M4의 프롬프트도 같은 원칙으로 재검토 필요.

### [2026-04-21] "내 파일로 해봐도 됨" 유도 문구 추가 필요
- **문제:** 모든 실습이 `samples/...` 고정 경로로 돼있어서 학습자가 "샘플로만 해야 하는 줄" 알 가능성. my-data/ 폴더가 있는데도 활용 유도가 약함.

### [2026-04-21] 실습은 한 번에 하나씩 공개 (전체 모듈 공통)
- **문제:** 각 모듈의 EXECUTE 섹션이 실습 1~7을 **한꺼번에 펼쳐 보여줌**. 학습자가 실습 1 하려고 스크롤 올리고, 결과 확인 후 실습 2 찾으러 또 스크롤 내리고... 스크롤 헬 발생.
- **왜 문제인가:**
  1. 비개발자 CSM은 채팅 UI가 익숙하지만 긴 문서 스크롤은 피로함.
  2. 실습 1을 하기도 전에 실습 5의 프롬프트가 눈에 들어오면 **정보 과부하**로 집중력 흐트러짐.
  3. 각 실습이 끝난 뒤에 다음 실습을 보여주면 "단계별 성취감"도 생김.
- **수정 방향:**
  1. EXECUTE 섹션 구조를 **"지금 이 실습 하나"만 노출**하는 방식으로 변경.
  2. 튜터 응답 패턴: `실습 N 프롬프트 안내` → 학습자 실행 → 결과/피드백 → `실습 N+1 프롬프트 안내`.
  3. WHY/EXPLAIN은 먼저 한 번에 보여주고 OK, EXECUTE만 단계식으로 쪼개기.
  4. references의 module-XX.md 파일은 전체 내용을 담되, **튜터가 대화에서 노출할 때는 1개씩** 꺼내는 패턴으로.
- **반영 위치:** SKILL.md의 2-Turn STOP 프로토콜에 "실습은 1회당 1개씩 노출" 규칙 추가하면 모든 모듈이 자동 적용됨.
